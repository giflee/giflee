如何将自己的页面部署在github上

Nov 3, 2015 | Hits

在很多时候可能都会看到一些托管在github上的代码都会有一个在线预览的地址，点进去发现域名为github.io自己的域名。这个功能在github上简直是前端工程师的利器，再也不用依托什么各种云来部署自己的项目了，简直是溜的不行。（注：强调了只是前端工程师，只能部署静态页面，服务端就算了，这上可没有mysql等等的环境）

1、申请github账户

有账户的略过，没账户的戳github官网直接注册。

2、安装git

因为github的操作都依赖于git，所以首先先要下载git。下载git下载图形界面客户端的不需要配置SSH了，如果打算使用命令行请Google下git ssh（强烈推荐使用命令行操作）

3、创建自己的repository

点击Create repositories，然后根据提示创建自己的库。创建完成后可以看到一些简单的指令。

4、使用基本的git指令上传到远端

这里选择你想部署到github上的项目，用命令行打开项目所在的路径
cd your_project

然后生成本地git库
git init
将本地库链接到远端
git remote add origin <server>
server为新建远端库的地址格式为.git结尾（在库的主页很好找）
添加文件到暂存区
git add .
提交commit到master
git commit -m "commit说明"
将项目推到远端
git push -u origin master

完成以上，OK最近本的git操作将你本地的项目放在github上托管，在github上查看库即可看到源代码。但是怎么在线去预览这个项目呢，这里就得知道github留下的一个小后门了。

5、部署到github生成预览地址

建立gh-pages分支
git checkout -b gh-pages
将gh-pages分支推到远端
git push origin gh-pages

ok,大功告成，现在可以访问刚才的项目了，在线地址为用户名.github.io/项目名/项目文件。具体的地址在查看库的设置选项里也能找到。
